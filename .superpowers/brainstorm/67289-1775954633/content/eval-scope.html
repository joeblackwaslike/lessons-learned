What scope are you targeting for this eval system?

This shapes everything — how much to build, how automated it needs to be, and whether it lives in CI or stays a dev tool.

A

Minimal viable signal

A rubric document + hand-crafted test transcripts. You run them manually, judge by eye. No code infrastructure. Fast to create — meant to validate directive quality before automating anything.

Pros

  • Ship in an hour
  • Zero maintenance burden
  • Forces careful thinking about what "pass" means

Cons

  • Tedious to re-run
  • Hard to compare across directive versions
B

Semi-automated

A script calls the Claude API with each scenario, collects responses, formats them for human review. Scoring is still manual but transcript generation is automated. Results written to evals/results/.

Pros

  • Reproducible runs
  • Easy to add new scenarios
  • Cheap to iterate on directive wording

Cons

  • Manual scoring is still a bottleneck
  • ~1–2 days to build runner
C

Fully automated pipeline

Automated transcript generation + LLM-as-judge scoring + structured results stored as JSON. Can run in CI or on demand. Comparable to eval infra in a production AI system.

Pros

  • Regressions caught automatically
  • Quantified scores over time
  • Scales to many directives

Cons

  • 3–5 days to build correctly
  • LLM-judge reliability needs calibration
  • Higher API cost per run

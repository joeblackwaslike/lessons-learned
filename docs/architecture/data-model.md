# Data Model

**Last updated:** 2026-04-01

---

## Overview

All data lives in `data/` as JSON files. There is no database.

| File                            | Purpose                                        | Edit directly?                              |
| ------------------------------- | ---------------------------------------------- | ------------------------------------------- |
| `lessons.json`                  | Source of truth — all lesson records           | Yes                                         |
| `lesson-manifest.json`          | Pre-compiled runtime manifest                  | No — run `lessons build`                    |
| `config.json`                   | Injection and scanning configuration           | Yes                                         |
| `cross-project-candidates.json` | T2 candidates awaiting review                  | No — generated by `lessons scan candidates` |
| `scan-state.json`               | Per-file byte offsets for incremental scanning | No — managed by scanner                     |

## Lesson record (`lessons.json`)

```jsonc
{
  "lessons": [
    {
      "id": "01JQSEED00000000000000001", // ULID, generated on add
      "slug": "pytest-tty-hanging-k9m2", // kebab-case summary + 4-char random suffix
      "summary": "pytest hangs in non-interactive envs due to TTY detection",

      // What went wrong and why (required, min 20 chars)
      "mistake": "Running bare `pytest` in Claude Code causes the process to hang...",

      // The correction (required, min 20 chars)
      "remediation": "Use `python -m pytest --no-header -p no:faulthandler`...",

      // Text injected into additionalContext — defaults to "## Lesson: {summary}\n{mistake}\n**Fix**: {remediation}"
      // Override for richer formatting or when the default is too long
      "injection": "## REQUIRED: pytest flags for Claude Code\n...",

      // If true, the tool call is denied rather than context-injected
      // blockReason is shown to Claude as the denial message (supports {command} placeholder)
      "block": true,
      "blockReason": "pytest command is missing required flags. Add `-p no:faulthandler --no-header`. Rerun as: `{command} -p no:faulthandler --no-header`",

      "triggers": {
        // Exact tool name match — fires on any use of this tool
        "toolNames": ["Bash"],

        // Regex patterns matched against the full Bash command string
        // Validated at add time — invalid regex is rejected
        "commandPatterns": ["\\bpytest\\b(?!.*(--no-header|-p no:faulthandler|TERM=dumb))"],

        // Glob patterns matched against Read/Edit/Write file paths
        "pathPatterns": [],

        // Reserved for future use
        "contentPatterns": [],

        // If true, inject at session start instead of PreToolUse
        // Reserved for reasoning reminders with no valid command/path trigger
        "sessionStart": false,
      },

      // Lesson scope — determines which projects receive this lesson
      // Omitted or { "type": "global" } = all projects
      // { "type": "project", "path": "/abs/path/to/project" } = that project only
      "scope": { "type": "global" },

      // 1–10; higher priority wins budget conflicts
      "priority": 8,

      // 0.0–1.0; lessons below config.minConfidence are excluded from manifest
      "confidence": 0.95,

      // true = excluded from manifest, surfaces in `lessons list` with a flag
      // Set automatically when confidence < 0.7 at add time
      "needsReview": false,

      // category:value taxonomy (e.g. "lang:python", "tool:pytest", "severity:hang")
      "tags": ["lang:python", "tool:pytest", "severity:hang"],

      // Session IDs where this mistake was observed (up to 5, for provenance)
      "sourceSessionIds": [],

      // How many times this pattern was seen across all scans
      "occurrenceCount": 0,

      "createdAt": "2026-03-29T00:00:00Z",
      "updatedAt": "2026-03-29T00:00:00Z",

      // sha256 of "mistake|remediation|JSON.stringify(triggers)"
      // Used for exact duplicate detection at add time
      "contentHash": "sha256:...",
    },
  ],
}
```

### Key field constraints

| Field                      | Constraint                                                                |
| -------------------------- | ------------------------------------------------------------------------- |
| `summary`                  | ≥ 20 chars, no `...` suffix, no template placeholders                     |
| `mistake`                  | ≥ 20 chars, no template placeholders                                      |
| `remediation`              | ≥ 20 chars, no template placeholders                                      |
| `triggers.commandPatterns` | Must be valid regex; invalid patterns are silently dropped at build time  |
| `triggers.sessionStart`    | Use sparingly — fires on every session startup, no dedup within a session |
| `confidence`               | < 0.7 → `needsReview: true`, excluded from manifest                       |
| `block` + `blockReason`    | Must be used together; `blockReason` supports `{command}` substitution    |

## Lesson manifest (`lesson-manifest.json`)

Generated by `lessons build`. This is the file the injection hook reads at runtime — not `lessons.json`.

```jsonc
{
  "type": "lessons-learned-manifest",
  "version": 1,
  "generatedAt": "2026-04-01T00:00:00Z",
  "config": {
    "injectionBudgetBytes": 4096,
    "maxLessonsPerInjection": 3,
    "minConfidence": 0.5,
    "minPriority": 1,
    "compactionReinjectionThreshold": 7,
  },
  "lessons": {
    "<ulid>": {
      "slug": "pytest-tty-hanging-k9m2",
      "priority": 8,
      "toolNames": ["Bash"],
      // Regex stored as { source, flags } for JSON-safe serialization
      // Reconstructed with new RegExp(source, flags) at match time
      "commandRegexSources": [{ "source": "\\bpytest\\b(?!...)", "flags": "" }],
      "pathRegexSources": [],
      "tags": ["lang:python", "tool:pytest", "severity:hang"],
      "injection": "## REQUIRED: pytest flags...",
      "summary": "pytest hangs in non-interactive envs due to TTY detection",
      "block": true,
      "blockReason": "...",
      "sessionStart": false,
      // Only present when scope.type === 'project'
      "projectPath": null,
    },
  },
}
```

Lessons are excluded from the manifest if `confidence < minConfidence`, `priority < minPriority`, or `needsReview: true`.

## Config (`config.json`)

```jsonc
{
  "injectionBudgetBytes": 4096, // Max bytes of additionalContext per tool call
  "maxLessonsPerInjection": 3, // Max lessons injected per tool call
  "minConfidence": 0.5, // Exclude lessons below this from manifest
  "minPriority": 1, // Exclude lessons below this from manifest
  "compactionReinjectionThreshold": 7, // Lessons above this priority re-inject after context compaction

  "scanPaths": ["~/.claude/projects/"],
  "autoScanIntervalHours": 24,
  "maxCandidatesPerScan": 50,

  "scoring": {
    "multiSessionBonus": 2, // Priority boost when seen in 2+ sessions
    "multiProjectBonus": 1, // Priority boost per additional project
    "hangTimeoutBonus": 1,
    "userCorrectionBonus": 1,
    "singleOccurrencePenalty": -1,
  },
}
```

## Candidate record (`cross-project-candidates.json`)

Written by `lessons scan candidates`. Read by `lessons scan promote` and `lessons review`.

```jsonc
{
  "generatedAt": "2026-04-01T00:00:00Z",
  "candidates": [
    {
      "index": 1, // 1-based, used by `lessons scan promote <index>`
      "tool": "Bash",
      "confidence": 0.75,
      "priority": 6,
      "occurrenceCount": 3,
      "sessionCount": 2, // Distinct sessions where this pattern was observed
      "projectCount": 1, // Distinct project directories
      "projects": ["jobsearch-tracker"],
      "mistake": "...",
      "remediation": "...",
      "tags": [],
      "trigger": "git stash", // Raw trigger text from the session
      "sourceSessionIds": ["..."],
      "signals": {
        "userCorrection": true, // Whether a user correction message was detected
      },
    },
  ],
}
```

**Scope inference from candidates:**

- `projectCount >= 2` → global lesson candidate
- `projectCount === 1` → project-specific lesson candidate

`lessons scan candidates` currently surfaces all recurring candidates. The project-specific subset (single-project candidates) will be surfaced via `lessons scan candidates --project` once project-scoped lessons are implemented.

## Scan state (`scan-state.json`)

```jsonc
{
  "files": {
    "/abs/path/to/session.jsonl": 184320, // Last byte offset read
  },
  "lastFullScanAt": "2026-04-01T00:00:00Z",
}
```

The scanner resumes each file from its saved offset, processing only new bytes. A `--full` flag resets all offsets to 0.

## Tag taxonomy

Tags follow `category:value` format. Established categories:

| Category    | Examples                                                         |
| ----------- | ---------------------------------------------------------------- |
| `lang`      | `python`, `typescript`, `javascript`, `go`                       |
| `tool`      | `pytest`, `git`, `npm`, `docker`                                 |
| `severity`  | `hang`, `data-loss`, `silent-failure`, `error`                   |
| `topic`     | `testing`, `auth`, `networking`, `types`                         |
| `candidate` | `node-gotchas-skill` — flagged for future skill file aggregation |
